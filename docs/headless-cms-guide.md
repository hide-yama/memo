# ヘッドレスCMSガイド — memoブログ向け検討資料

## ヘッドレスCMSとは

従来のCMS（WordPressなど）は「管理画面＋表示画面」がセットになっている。ヘッドレスCMSは**管理画面（コンテンツ管理）だけ**を提供し、表示はNext.jsなど好きなフレームワークで自由に作る仕組み。

```
【今のmemo】
Markdownファイル → Next.js → Vercel → 読者

【ヘッドレスCMS導入後】
CMS管理画面で執筆 → API → Next.js → Vercel → 読者
```

**メリット:** ブラウザから記事を書ける、スマホからも編集できる、下書き・プレビュー・予約投稿などがUIで完結する
**デメリット:** API依存になる、無料枠の制限がある、移行コストがかかる

---

## memoブログとの相性

現状のワークフロー（Markdown + git push）は軽量で高速。ヘッドレスCMSを入れるべきタイミングは:

- 記事数が30本を超えてファイル管理がつらくなったとき
- ブラウザやスマホから書きたくなったとき
- 複数人で運用する可能性が出てきたとき

**今すぐ必要ではないが、選択肢として知っておく価値はある。**

## 候補の比較（無料プラン）

### 1. microCMS（国産・最有力）

| 項目 | 内容 |
|------|------|
| 月額 | **0円**（Hobbyプラン） |
| 記事数 | 10,000件 |
| API数 | 5つ（ブログ・カテゴリ・タグ等に十分） |
| 転送量 | **20GB/月**（月6,000〜7,000PV相当） |
| メンバー | 3人 |
| 日本語UI | あり（完全日本語） |
| 日本語サポート | あり |
| 次の有料プラン | 4,900円/月 |

**良い点:**
- 国産で日本語UIが完璧。ドキュメントもコミュニティも日本語
- Next.js公式チュートリアルあり、SDKも用意されている
- リッチテキストエディタで日本語入力に問題なし
- メディア（画像）は無料プランでも容量無制限

**注意点:**
- 転送量20GB/月を超えるとAPIが停止する（個人ブログ初期なら余裕）
- 無料プランではPOST API（外部からの書き込み）不可
- 無料→有料の差が0円→4,900円と大きい

**memoとの相性: ★★★★★** — 国産・Next.js連携が最も楽・個人ブログに最適

---

### 2. Notion（既にNotionユーザーなら）

| 項目 | 内容 |
|------|------|
| 月額 | **0円**（個人利用） |
| 記事数 | 実質無制限 |
| API制限 | 3リクエスト/秒（レートリミット） |
| 転送量 | 制限なし（Notion側） |
| 日本語UI | あり |
| 次の有料プラン | $10/月 |

**良い点:**
- 完全無料で記事数の制限がほぼない
- Notionのエディタがそのまま使える（使い慣れていれば最強）
- スマホアプリからも編集可能

**注意点:**
- **画像のURLが1時間で失効する**（S3の署名付きURL）。自前で画像を保存し直す処理が必要
- APIは1回100ブロックまで。長い記事はページネーション処理が必要
- CMS用に設計されていないので、カスタムコードが多くなる
- Webhookがないのでリアルタイム更新ができない

**memoとの相性: ★★★☆☆** — 無料だが画像問題と開発コストがネック

---

### 3. Sanity（海外製・無料枠が大きい）

| 項目 | 内容 |
|------|------|
| 月額 | **$0** |
| ドキュメント数 | 10,000件 |
| API CDNリクエスト | **500万回/月** |
| APIリクエスト | 100万回/月 |
| 転送量 | **100GB/月** |
| メンバー | 20人 |
| 日本語UI | なし（プラグインで一部対応） |
| 次の有料プラン | $15/ユーザー/月 |

**良い点:**
- 無料枠が圧倒的に大きい（転送量100GB、CDN 500万回）
- 管理画面（Sanity Studio）をReactでカスタマイズできる
- リアルタイムコラボレーション機能あり
- Next.js公式連携パッケージ（`next-sanity`）がある

**注意点:**
- **日本語のUIもドキュメントもサポートもない**
- GROQという独自クエリ言語を覚える必要がある
- 初期セットアップの学習コストが高い
- 日本での利用者コミュニティが小さい

**memoとの相性: ★★☆☆☆** — スペックは魅力的だが日本語環境の弱さが大きい

---

### 4. Contentful（海外製・大手）

| 項目 | 内容 |
|------|------|
| 月額 | **$0** |
| レコード数 | 10,000件 |
| APIコール | 10万回/月 |
| 転送量 | 50GB/月 |
| 日本語UI | なし |
| 次の有料プラン | **$300/月** |

**注意点:**
- APIコール10万回/月の制限がやや厳しい
- 無料→有料が$300/月と個人には非現実的
- 日本語UI・サポートなし

**memoとの相性: ★☆☆☆☆** — 個人ブログにはオーバースペックかつ割高

## 結論：もし導入するなら

| 優先度 | 選択肢 | 理由 |
|--------|--------|------|
| **第1候補** | **microCMS** | 国産・日本語完備・Next.js連携が最も簡単・無料で始められる |
| 第2候補 | Notion | 完全無料だが画像問題と開発コストあり。Notionヘビーユーザー向け |
| 見送り | Sanity / Contentful | 日本語ブログとの相性が悪い |

## 導入しない場合でも得られるもの

ヘッドレスCMSを入れなくても、以下はMarkdown運用のまま実現できる:

- **カテゴリ分け** → frontmatterに`category`フィールドを追加するだけ
- **公開日スケジューリング** → `date`が未来の記事を非表示にするロジック1行
- **下書き管理** → `published: false`（既に実装済み）

CMSが本当に必要になるのは「ブラウザから書きたい」「スマホから編集したい」と感じたとき。それまではMarkdown運用で十分。
